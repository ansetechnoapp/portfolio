🖼️ Démonstration d'Optimisation d'Images

Cette page démontre les différentes méthodes d'optimisation d'images disponibles dans ce projet Astro.

1. Composant OptimizedImage (Recommandé)

Utilise automatiquement les images optimisées si disponibles, sinon utilise le composant Image d'Astro.

Image de test optimisée :

Image at-work optimisée :

Voir le code 
<OptimizedImage
  src="/assets/test-image.jpg"
  alt="Image de test optimisée automatiquement"
  width={400}
  height={267}
  class="rounded-lg shadow-md"
  loading="lazy"
/>

2. Composant Image d'Astro

Optimisation automatique à la volée par Astro avec Sharp.

Image optimisée par Astro :

Voir le code 
<Image
  src="/assets/test-image.jpg"
  alt="Image optimisée par Astro"
  width={400}
  height={267}
  format="webp"
  quality={80}
  class="rounded-lg shadow-md"
  loading="lazy"
/>

3. Comparaison des Performances

📊 Métriques d'optimisation :

  • Format AVIF : ~50-60% plus petit que JPEG
  • Format WebP : ~25-35% plus petit que JPEG
  • Images responsives : 60-80% de réduction sur mobile
  • Lazy loading : Améliore le temps de chargement initial

✅ Avantages du système actuel :

  • Optimisation automatique lors du build
  • Formats modernes avec fallbacks
  • Images responsives automatiques
  • Compatible avec tous les navigateurs
  • Optimisé pour Vercel

4. Bonnes Pratiques

✅ À faire :

  • Toujours spécifier alt, width, height
  • Utiliser loading="lazy" sauf above-the-fold
  • Optimiser les images source avant ajout
  • Tester sur différents appareils
  • Utiliser OptimizedImage par défaut

❌ À éviter :

  • Images sans attribut alt
  • Dimensions non spécifiées
  • Images trop lourdes en source
  • Loading eager sur toutes les images
  • Formats obsolètes uniquement
